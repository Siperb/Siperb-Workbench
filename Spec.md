# Siperb Workbench

# Features:
1. Build siperb and phone html
2. Allows local files when building the scripts/css/html
3. Accepts folders or files which it will run each test.
4. Tests are written such that it is 'added in' the html
5. Results are generated for each 'suite' (folder name) each folder get its own results with details of each file or test run, the root  gets tha accumalation of all the results.
6. Can run headless but still is in a browser environment
7. Can run using the test-bench.html which will show the tests in realtime with the iframe on the right side, testing info and logs on left. 
8. Provisioning can be done with Session_ID, User_ID,Device_ID or PAT(Pesonal Access Token)
